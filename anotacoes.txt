0. criar a estrutura inicial:
npx sequelize init

obs: esse comando cria a pasta models + arquivo index, a pasta migrations e a pasta seeds; cria também o arquivo config.json.
se já existirem as pastas ou arquivos, o comando não irá sobrepôr.
se for modificar o config.json terá que alterar no models/index.js também.

1. preparar o arquivo config/config.js (ou .json).

2. criar o DB:
npx sequelize db:create

3. a responsabilidade de criar as tabelas é das migrations:
npx sequelize migration:generate --name cria-tabela-tal

4. colocar na migration as informações sobre as colunas e suas restrições (em up/criar e down/desfazer).

5. executa a migration depois de montada, toda prontinha e linda (criar as tabelas):
npx sequelize db:migrate

obs. para desfazer o último comando de migration
npx sequelize db:migrate:undo

6. popular as tabelas com as seeds
npx sequelize db:seed:all

7. criar o model

7.1 ele é uma função com dois parâmetros: sequelize & DataType

7.2 tem que chamar o método sequelize.define("nome_do_modelo", { propriedades: DataTypes.tipos })

obs: é muito parecido com a migration (sequelize.tipo)
